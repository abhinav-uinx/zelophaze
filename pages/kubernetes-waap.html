Kubernetes-Native WAAP

Secure your containerized microservices from inside the cluster. Embedded eBPF sidecars and native Envoy ingress integration inspect both North-South and East-West traffic with zero architectural modification.

Why It Is Necessary: The East-West Blind Spot

Traditional edge WAFs only inspect ingress traffic (North-South) entering from the internet. Once an attacker bypasses the perimeter via a compromised dependency or SSRF vulnerability, they have unrestricted lateral access (East-West) across unencrypted Kubernetes pods. Furthermore, legacy security appliances break GitOps CI/CD pipelines because external security teams must manually approve routing changes. A cloud-native WAAP must live natively inside the cluster orchestration fabric.

Core Architectural Functions

How Zelophaze integrates directly into modern container orchestrators.

eBPF & Envoy Sidecar Acceleration

Operates at kernel level using eBPF hooks and native Envoy filter plugins. Inspects RPCs, gRPC, and REST calls directly inside the Linux kernel memory space without proxy overhead.

  • Sub-millisecond pod-to-pod inspection
  • Zero application code modification
  • Ultra-low memory & CPU footprint

GitOps Declarative CRDs

Security rules are defined as Kubernetes Custom Resource Definitions (CRDs). Developers commit security policies alongside application manifests in Git repositories for automated enforcement.

  • 100% Infrastructure-as-Code compatibility
  • Automated versioning & rollback via ArgoCD
  • Per-namespace tenant isolation

Auto-Scaling Dynamic Pod Defense

As Kubernetes Horizontal Pod Autoscalers (HPA) spin up thousands of new pod replicas during traffic spikes, Zelophaze security nodes instantly attach and inherit active defense policies within milliseconds.

  • Instantaneous replica security synchronization
  • Dynamic IP pool discovery
  • Automated mutual TLS (mTLS) verification

Lateral SSRF & Exploit Containment

Continuously maps runtime service dependencies. If a web pod attempts unauthorized communication with an internal database or metadata API (AWS IMDSv2), the connection is instantly severed.

  • Zero-trust service mesh boundary
  • Real-time anomaly containment
  • Automated forensic container snapshot
